## 이유 (사용자 선택 근거 추정)

- 다른 이전 경험에 풀스택 백그라운드가 있어 일관성 있는 시그널
- Android만 강조 시 "또 똑같은 안드로이드 토이 앱" 인상 회피
- ADR-006 백엔드 프록시 + 보안 보강 결정이 자연스럽게 메인 스토리화

## 보존되는 것 (시그널 변경에도 유지)

- Health Connect 통합 (StepSourceResolver) — 보조 차별화로 유지, 메인은 아님
- Naver Geocoding 절충안 (ADR-006) — **보안 컨텍스트로 재포지셔닝** (OWASP M3 백엔드 프록시 패턴)
- 다른 5개 면접 답변 (Compose, 센서/배터리, FGS Type, 테스트/CI) — 변경 없음
- 디자인 시스템 (Material 3 colors/themes) — 변경 없음

## 기각되는 것 (메인 어필에서 제외)

- "한국 도메인 사고력" 메인 셀링 포인트 → 보조로 격하
- "한국 사용자 관점에서..." 톤의 README 첫 단락 → 풀스택 톤으로 교체
- 외국 면접관 영문 키워드 (interview-prep.md #6에 있던 것) → 백엔드 영문 키워드 추가 권장

## 면접 시 새 약점 (대비 필요)

옵션 C는 다음 공격을 받을 수 있음 — `interview-prep.md #1 보강` 답변으로 방어:

1. **"왜 안드로이드 신입이 풀스택?"** → 답변 작성 완료
2. **"둘 다 어설픈 거 아닌가요?"** → 의도적 범위 축소 + ADR로 방어
3. **"백엔드 회사 가시지"** → 안드로이드 본업 + 백엔드 컨텍스트 이해

## 검증 트리거 (재고 시점)

다음 중 하나 발생 시 시그널 재선택:
- 워크온 JD에서 "백엔드 무관" 명시 발견 → 옵션 B (Android 깊이)로 전환
- 외국계도 함께 지원하기로 결정 → 옵션 B 또는 D
- 백엔드 코드 실행 검증 실패 (Render 배포 안 됨) → 풀스택 주장 근거 약화 → 시그널 변경 고려

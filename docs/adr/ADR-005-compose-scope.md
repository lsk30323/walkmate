# ADR-005 — Compose 도입 범위 (홈 통계 카드 1화면만)

- **상태**: Accepted
- **작성일**: 2026-04-25 (Week 8)
- **결정자**: lsk12
- **연관**: 명세서 §2(UI), §7 ⑤ 면접 답변 #2

## 1. 컨텍스트

신입 포트폴리오에서 Compose 미사용은 약점. 풀 Compose 전환은 학습 비용·안정성 리스크.

## 2. 고려한 옵션

### 옵션 A — 풀 Compose 전환
- 장점: 모던 시그널
- 단점: 학습 비용 ★★★★, 일정 파탄

### 옵션 B — 홈 통계 카드 1화면만 ⭐
- 장점: "양쪽 다 만져봤다" 시그널, 학습 비용 1일
- 단점: "왜 다 안 했어요?" 면접 질문 받을 수 있음

### 옵션 C — Compose 미사용
- 장점: 안정성
- 단점: 신입 포트폴리오 평균 미달 → 감점

## 3. 결정

**옵션 B (1화면)** 채택.

## 4. 근거

- 10주 일정 흡수 가능 (Week 8에 +1일)
- 면접 답변 #2 ("XML 메인 + Compose 보조")에 직접 활용
- `MPAndroidChart`를 `AndroidView`로 감싸는 점진 전환 패턴 검증 가능

## 5. 결과

### 긍정적
- 신입 포트폴리오 평균 이상 시그널
- 점진 전환 경험 어필

### 부정적
- 면접관이 "왜 풀 전환 안 했어요?"라고 압박 가능 — 답변 준비 필수

### 재검토 트리거
- v1.1에서 풀 Compose 전환 검토

# ADR-000 — WalkMate 프로젝트 착수 결정

- **상태**: Accepted
- **작성일**: 2026-04-25 (Week 1 초)
- **결정자**: lsk12
- **연관**: 명세서 §1, walkmate-review-FINAL.md

## 1. 컨텍스트

스왈라비/워크온 안드로이드 신입 채용을 목표로 포트폴리오를 만들어야 한다. 3가지 주제 후보:
- (A) WalkMate (만보기 + GPS + 그룹 랭킹) — 도메인 정합 ★★★★★
- (B) 일반 SNS — 도메인 정합 ★★ (만들기 쉬움)
- (C) 학습 기록 앱 — 도메인 정합 ★★★

## 2. 고려한 옵션

### 옵션 A — WalkMate
- 장점: 도메인 직결, 센서·GPS·Foreground Service 등 JD 키 항목 직접 증명
- 단점: 워크온 본 서비스와 흡사 → "복제" 리스크
- 공수: 10주 (재조정 후)

### 옵션 B — 일반 SNS
- 장점: 만들기 쉬움, 일정 안전
- 단점: 워크온 도메인 어필 불가능, 차별화 ★

### 옵션 C — 학습 기록 앱
- 장점: 차별화 가능
- 단점: 헬스케어 도메인 어필 부족

## 3. 결정

**옵션 A (WalkMate)** 채택.

## 4. 근거

- 도메인 직결도가 채용 의사결정에 결정적
- 복제 리스크는 차별화 각도(Health Connect, 재부팅/타임존 baseline)로 방어 가능
- 차별화 답변 스크립트 (interview-prep.md #1)로 면접 공격 대응

## 5. 결과

### 긍정적
- JD 매칭 표면적 ↑
- 면접 시 기술 디테일 풍부 (센서, FGS Type, baseline 등)

### 부정적
- 복제 리스크 답변 스크립트 필수 — 준비 안 하면 자살골
- 워크온 본 앱 미사용 시 "써보지도 않은 채로 만들었나" 의심 → 본 앱 사용 경험 확보 필수

### 재검토 트리거
- 워크온 본 앱이 곧 Health Connect 지원하면 차별화 각도 약화 → 재검토
- 채용공고에 "Wear OS" 요구 발견 시 스코프 조정
